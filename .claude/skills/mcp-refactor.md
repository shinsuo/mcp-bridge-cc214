# MCP Refactor — 代码重构审计

审计代码质量问题，输出问题清单，经用户确认后逐步清理。

## 重构铁律

| # | 规则 | 说明 |
|---|------|------|
| 1 | 零功能变更 | 不改变任何业务逻辑，只改善代码质量 |
| 2 | 编译必过 | 每轮重构后执行 `npm run build` |
| 3 | 问题先列后改 | 先完成审计清单，用户确认后才改代码 |

## 执行指令

### 1. 审计（≥ 5 次搜索/读取）
- 扫描用户指定的目标目录/文件
- 对照 `.agent/rules/project-rules.md` 检查违规项
- 所有反馈使用**简体中文**

### 2. 问题清单
审计完成后**先输出清单再动代码**：
```
| # | 严重度 | 文件 | 行号 | 问题描述 | 建议修复 |
|---|--------|------|------|---------|---------|
| 1 | 🔴 高 | xxx.ts | L42 | 死代码未清理 | 删除 |
| 2 | 🟡 中 | yyy.ts | L18 | 重复代码 | 提取共享函数 |
| 3 | 🟢 低 | zzz.ts | L95 | 命名不规范 | 重命名 |
```

**严重度**：
- 🔴 高：违反规范硬性规则
- 🟡 中：代码质量问题（重复代码、过时模式、缺失注释）
- 🟢 低：风格建议（命名、格式、微优化）

**输出清单后暂停，等待用户确认。**

### 3. 重构执行
用户确认后按严重度从高到低执行：
- 提取重复代码为共享函数
- 补充 JSDoc 注释
- 清除死代码（未使用的导入、变量、函数）

### 4. 编译验证
- 执行 `npm run build`，失败则立即修复

### 5. 完成汇报
```
| # | 文件 | 改动类型 | 改动说明 | 功能影响 |
|---|------|---------|---------|---------|
| 1 | xxx.ts | 代码清理 | 删除死代码 | 无 |
```

最后声明：**"以上改动均不涉及功能逻辑变更，仅为代码质量改善。"**
