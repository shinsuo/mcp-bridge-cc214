# MCP Execute — 计划执行

按 `specs/[feature_name]/plan.md` 中的复选框顺序逐步执行实施。

## 执行铁律

| # | 规则 | 说明 |
|---|------|------|
| 1 | 逐步执行 | 按 plan.md 复选框顺序逐个执行，不跳步 |
| 2 | 编译必过 | 所有代码修改完成后执行 `npm run build` |
| 3 | 状态同步 | 每完成一步，将 plan.md 中对应项改为 `- [x]` |
| 4 | 失败即停 | 编译失败立即修复并重试，不跳过 |
| 5 | 禁止偏离 | 不执行 plan.md 中没有的改动，发现遗漏报告用户 |

## 执行指令

### 1. 阅读计划
- 打开 `specs/[feature_name]/plan.md`
- 定位分步实施章节的复选框
- 理解所有步骤范围和依赖关系

### 2. 逐步执行
- 按顺序执行所有 `- [ ]` 项
- 无依赖的步骤可并行执行
- 每次修改后 1-2 句话说明改动理由
- plan.md 代码片段与实际文件冲突时，以实际文件为准并报告偏差

### 3. 编译验证（强制）
- 执行 `npm run build`
- 失败则：读报错 → 定位 → 修复 → 重新 `npm run build` → 循环直到通过
- 通过后将编译步骤标记为 `- [x]`

### 4. 状态同步
- 确认 plan.md 所有 `- [ ]` 变为 `- [x]`
- 未完成的步骤标注原因

### 5. 完成汇报
输出汇报表：
```
| 阶段 | 步骤 | 状态 | 备注 |
|------|------|------|------|
| A | 步骤描述 | ✅/❌ | 偏差说明 |
| B | npm run build | ✅/❌ | 编译日志 |
| C | UPDATE_LOG.md | ✅/❌ | — |
```

完成后提醒用户：
- 在 Cocos Creator 中重新加载插件验证效果
- 如需进一步验证，可使用 `/mcp-verify` 技能
