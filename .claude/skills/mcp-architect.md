# MCP Architect — 实施计划创建

基于 `specs/[feature_name]/spec.md` 创建实施计划 `specs/[feature_name]/plan.md`。

## 交付标准（7 项硬性指标）

| # | 检查项 | 合格标准 |
|---|--------|----------|
| 1 | 文件清单表格 | 含所有涉及文件、所属层级、改动性质的 Markdown 表格 |
| 2 | 分步实施复选框 | 每个步骤使用 `- [ ]` 格式 |
| 3 | 阶段分组 | 步骤按阶段分组（A: 代码修改 → B: 编译验证 → C: 文档更新） |
| 4 | 层级标签 | 每个步骤标注 `[Main]` / `[Scene]` / `[Build]` / `[Docs]` |
| 5 | 编译验证步骤 | 必须包含 `npm run build` 步骤 |
| 6 | 文档更新步骤 | 必须包含 `UPDATE_LOG.md` 更新步骤 |
| 7 | 代码片段 | 关键改动步骤附带改动前 → 改动后代码对比 |

## 执行指令

### 1. 阅读规范
- 加载 `specs/[feature_name]/spec.md`
- 提取：根因分析结论、修复方案细节、涉及文件清单、边界情况

### 2. 起草计划
仅创建/更新 `specs/[feature_name]/plan.md`，使用**简体中文**。

### 3. 文档结构

#### 架构设计
- **文件清单表格**：含层级（Main/Scene/Build/Docs）、改动性质、一句话说明
- **架构影响评估**：无变更时用 `> [!NOTE]` 声明
- **关键流程图**：多组件交互时用 ASCII 图

#### 分步实施
按阶段分组：
```
### 阶段 A: 代码修改
- [ ] [Main] 步骤描述...

### 阶段 B: 编译验证
- [ ] [Build] 执行 `npm run build`

### 阶段 C: 文档更新
- [ ] [Docs] 更新 `UPDATE_LOG.md`
```

### 4. 自检
创建 plan.md 后输出自检表：
```
| 检查项 | 要求 | 实际 | 达标 |
|--------|------|------|------|
| 文件清单表格 | 有 | 有/无 | ✅/❌ |
| 复选框步骤数 | ≥ 3 | ? | ✅/❌ |
| 阶段分组 | 有 | 有/无 | ✅/❌ |
| 层级标签 | 有 | 有/无 | ✅/❌ |
| 编译验证步骤 | 有 | 有/无 | ✅/❌ |
| 文档更新步骤 | 有 | 有/无 | ✅/❌ |
| 代码片段 | ≥ 1 | ? | ✅/❌ |
```
